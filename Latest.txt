PHASE 7 — Full Deployment Clarification (Local + Production Handoff)

Local development stack:
1. Start infra services from the repository root:
  - `docker-compose up -d`
2. Start the Spring Boot backend:
  - `./mvnw spring-boot:run`
3. Configure the frontend environment file:
  - `frontend/.env.local` with `NEXT_PUBLIC_API_URL=http://localhost:8080`
4. Start the Next.js frontend:
  - `cd frontend && npm run dev`

Required runtime dependencies:
- PostgreSQL 17
- Redis 7
- RabbitMQ 4-management
- Spring Boot backend on port 8080
- Next.js frontend on port 3000

Production deployment clarification:
1. Backend must be deployed first so the frontend has a stable API base URL.
2. Frontend must point `NEXT_PUBLIC_API_URL` to the public backend URL at build time.
3. Build validation for release:
  - Backend: `./mvnw -q -DskipTests compile`
  - Frontend: `npm run build` inside `frontend/`
4. Do not hardcode any Railway, Vercel, or other host URL in source code.
5. Keep `.env.local` out of version control; commit only `frontend/.env.example`.

Known deployment note:
- On this Windows host, Next.js SWC native bindings are invalid, but the frontend build succeeds using webpack fallback. That is a local environment issue, not a code defect.
