ARCHITECTURE DECISIONS - Locked (no deviations):
- Architecture        : Modular monolith. Each domain independently extractable to microservice.
- Dependency Injection: Constructor injection via @RequiredArgsConstructor. All fields private final.
- Time handling       : java.time.Instant (UTC) for ALL timestamp fields. NEVER LocalDateTime.
- DB schema time type : TIMESTAMPTZ (not TIMESTAMP without zone) in all Flyway migrations.
- Role column type    : PostgreSQL ENUM (not VARCHAR). CREATE TYPE user_role AS ENUM ('ATTENDEE','ORGANIZER','ADMIN').
- Soft delete         : deleted_at TIMESTAMPTZ on bookings. Never hard-delete booking records.
- Optimistic locking  : @Version Long version on Booking and TicketTier JPA entities.
- Redis distributed lock: Atomic SET key value NX EX ttl. Lua script for ownership-checked release.
- Inventory split     : Redis is fast cache layer. PostgreSQL is source of truth. Both updated transactionally.
- Stripe webhook      : HTTP 200 returned ONLY after DB transaction commits. Controller NOT @Transactional.
- Webhook idempotency : UNIQUE constraint on processed_stripe_events.stripe_event_id.
                        Concurrent duplicate delivery -> DataIntegrityViolationException -> silently ignore.
- QR code generation  : ASYNC via ticket.generation.queue RabbitMQ queue. NEVER inline in webhook handler.
- State machine       : @EnableStateMachineFactory (one factory, instance per booking request).
                        NEVER @EnableStateMachine (that creates a single shared instance - wrong).
- Scheduler safety    : @Scheduled jobs must acquire a distributed lock (TTL 25s) before executing.
                        This prevents duplicate execution across Railway replicas.
- Logging             : SLF4J only. Parameterized messages. X-Correlation-ID in MDC, propagated in headers.
- Business constants  : All hardcoded values live in BusinessConstants.java (com.ticketing.common.util).
- Transactions        : @Transactional(readOnly = true) at class level on service classes.
                        Override with @Transactional (no readOnly) on write methods.
- Authorization       : @PreAuthorize on every sensitive controller method. No security by obscurity.
